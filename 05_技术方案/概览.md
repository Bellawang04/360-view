## 开发前提

进入 Eclipse ADT，连上测试系统后，先确认以下表有数据：

| 表名 | 用途 | 验证方式（SE16N） |
|------|------|----------------|
| `BSID` | 未清AR凭证 | WHERE BUKRS = 'CN01' |
| `BSAD` | 已清AR凭证 | 同上 |
| `KNKK` | 客户信用管理 | WHERE MANDT = 当前集团 |
| `VBAK` | 销售订单头 | WHERE VKORG = 'CN01' |
| `VKDFS` | 合同条款 | 按实际查 |
| `KONA` | 返利协议头 | 按实际查 |

## 开发顺序

```
第1步  执行 01_Z表DDL.md 的 DDL → 建表 Z_CUST_KPI_CACHE
       （SE11 建表，或 ADT 新建 Database Table）

第2步  依次创建5个 CDS View（02~06）
       每个 View 建完后用 F8 预览数据，对照 SE16N 验证数值正确

第3步  创建批处理 ABAP 程序（07）
       SE38 建程序，手动执行一次，SE16N 查 Z_CUST_KPI_CACHE 有数据

第4步  SM36 注册每日定时任务，设定每天 00:00 自动执行
       SM37 次日早上查执行日志确认成功
```

## 命名规范（Z开头，防止和SAP标准冲突）

| 类型 | 命名格式 | 示例 |
|------|---------|------|
| Z 表 | `Z_CUST_KPI_CACHE` | 缓存表 |
| Z 表（外部风险） | `Z_CUST_RISK_EXT` | Phase 1 手动写入演示数据 |
| CDS View | `Z_C_` + 规则缩写 | `Z_C_AR_AGING_ALERT` |
| ABAP 程序 | `Z_CUST_KPI_BATCH` | 批处理主程序 |
| Job 名称 | `Z_CUST_360_DAILY` | SM36 中的 Job 名 |

## 注意事项

1. **货币字段**：CURR 类型必须配参考字段（`@Semantics.amount.currencyCode`），建表时要注意
2. **CDS View 激活**：每次修改 View 后要重新激活（Ctrl+F3），否则查询的是旧版本
3. **第2周对齐**：`Z_CUST_KPI_CACHE` 建好后立即发表结构给智能分析组确认，他们的 Job 也要往这张表写
4. **BSID vs BSAD**：AR 账龄逻辑只读 `BSID`（未清项），`BSAD` 是已清项不计入逾期
