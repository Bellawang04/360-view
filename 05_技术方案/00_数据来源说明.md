# 数据来源说明

> 智能提醒组所有规则的数据均来自 SAP S/4HANA 内部表  
> 查询工具：SE16N（直接查表）/ VA05、FBL5N 等业务事务码

---

## 一、数据层级总览

```
第一层：SAP 内部业务数据（核心，直接读取）
  └── 财务：BSID / BSAD / KNKK
  └── 销售：VBAK / VBAP / VEDA / KNA1
  └── 返利：KONA / KONH

第二层：外部数据（Phase 1 手动演示，Phase 2 接企查查 API）
  └── 工商状态 / 司法风险 / 税务处罚
  └── Phase 1 手动写入 Z_CUST_RISK_EXT

第三层：计算产生的数据（你们自己造）
  └── 每日批处理 Job 读第一层 → 计算 → 写入 Z_CUST_KPI_CACHE
  └── AI 风险摘要：调用 BTP LLM 实时生成，不持久化
```

---

## 二、各规则对应的源数据表

### 规则1：AR账龄逾期 → 读 `BSID`

| 字段 | 含义 | 说明 |
|------|------|------|
| `KUNNR` | 客户编号 | 关联客户 |
| `BUKRS` | 公司代码 | 区分法人主体 |
| `DMBTR` | 凭证金额（本币） | 应收金额 |
| `ZFBDT` | 净到期日 | **账龄计算用这个**，不是凭证日期 |
| `SHKZG` | 借贷标志 | `S`=借方（应收），只取 S |
| `KOART` | 科目类型 | `D`=客户科目，只取 D |
| `WAERS` | 货币单位 | |

> `BSID` = 未清项（未付款），`BSAD` = 已清项（已付款）  
> 账龄逾期只读 `BSID`

**SE16N 验证：**
```
表：BSID
条件：BUKRS = CN01，KOART = D，SHKZG = S
查看：ZFBDT 字段，比今天早的就是逾期
```

### 规则2：信用额度预警 → 读 `KNKK`

| 字段 | 含义 | 说明 |
|------|------|------|
| `KUNNR` | 客户编号 | |
| `KKBER` | 信用控制范围 | 一个客户可能有多条，取占用率最高 |
| `KLIMK` | 信用额度上限 | = 0 表示未设置，不触发预警 |
| `SKFOR` | 当前已占用额度 | 含未清发票 + 未交货订单金额 |
| `WAERS` | 货币单位 | |

> 占用率 = `SKFOR / KLIMK × 100`  
> `KLIMK = 0` 标记为 NA，不预警

**SE16N 验证：**
```
表：KNKK
条件：KUNNR = 你的测试客户
查看：KLIMK 和 SKFOR 的值，手动算占用率确认
```

---

### 规则3：长期未下单 → 读 `VBAK`

| 字段 | 含义 | 说明 |
|------|------|------|
| `KUNNR` | 售达方客户 | |
| `VKORG` | 销售组织 | |
| `AUDAT` | 订单创建日期 | **取 MAX，算最近下单日** |
| `AUART` | 订单类型 | 需上机确认实际值（OR / ZOR 等） |
| `ABGRU` | 拒绝原因 | 不为空 = 被拒绝，排除掉 |
| `NETWR` | 订单净值 | 用于销售活跃度趋势图 |

> 未下单天数 = 今天 - MAX(AUDAT)  
> 从没下过单的客户在 VBAK 中无记录，批处理 Job 需特殊处理（写 DAYS_INACTIVE = 9999）

**SE16N 验证：**
```
表：VBAK
条件：VKORG = CN01
先看 AUART 有哪些值 → 确定 WHERE 条件应该写什么
```

---

### 规则4：合同快到期 → 读 `VEDA` / `VBAK`

| 表 | 字段 | 含义 | 说明 |
|----|------|------|------|
| `VBAK` | `AUART` | 合同类型 | KM / ZKM 等，需确认 |
| `VBAK` | `KUNNR` | 客户 | |
| `VEDA` | `VBELN` | 合同编号 | 关联 VBAK |
| `VEDA` | `VDATU` | 合同开始日 | |
| `VEDA` | `VBDAT` | **合同结束日** | 到期日用这个 ★ |

> `VEDA` 是合同专用表，存有效期  
> 剩余天数 = `VBDAT` - 今天（负值 = 已过期）

**SE16N 验证：**
```
第一步：表 VBAK，条件 AUART = KM（或其他合同类型）
        找到一张合同的 VBELN 编号

第二步：表 VEDA，条件 VBELN = 上面找到的编号
        查看 VBDAT 字段是否有到期日
        如果 VEDA 无数据，改查 VBKD 表的 KDATB 字段
```

---

### 规则5：年度返利协议未签 → 读 `KONA`

| 字段 | 含义 | 说明 |
|------|------|------|
| `KUNNR` | 客户编号 | |
| `VKORG` | 销售组织 | |
| `KNUMA_BO` | 返利协议编号 | |
| `BOTYP` | 协议类型 | BO01/BO02 等，需确认 |
| `BOSTA` | 协议状态 | `B`=已激活，`A`=草稿，`C`=已结算 |
| `DATAB` | 有效期开始 | |
| `DATBI` | 有效期结束 | |

> 判断逻辑：当前财年内，`BOSTA = 'B'` 且有效期覆盖当前日期 = 已签  
> 无记录 或 只有草稿状态 = 未签（UNSIGNED）

**SE16N 验证：**
```
表：KONA
条件：KUNNR = 你的测试客户，VKORG = CN01
查看：BOTYP 有哪些类型值，BOSTA 的实际状态
```

## 三、外部风险数据（Phase 1 演示）

Phase 1 不接真实 API，手动写入演示数据：

**表名：`Z_CUST_RISK_EXT`**（需自行建表，字段参考如下）

| 字段 | 含义 | Phase 1 值 |
|------|------|-----------|
| `KUNNR` | 客户编号 | |
| `RISK_LEVEL` | 风险等级 | 0=正常，1=关注，2=预警，3=高危 |
| `BUSSINESS_ST` | 工商状态 | 正常营业 / 注销 / 吊销 |
| `JUDICIAL_RISK` | 司法风险 | 无 / 有执行记录 |
| `DATA_SOURCE` | 数据来源 | 演示数据（固定写死） |
| `UPDATE_DATE` | 更新日期 | 手动填写日期 |

> Phase 2：通过 BTP Integration Suite 调用企查查 API，每日自动拉取，替换手动数据

## 四、计算结果缓存表

**表名：`Z_CUST_KPI_CACHE`**（详见 [[01_Z表DDL]]）

- 不是原始数据，是批处理 Job 每日计算后写入的**结果表**
- 页面读这张表展示，不直接读源表（保证响应速度 < 1秒）
- 提醒组写入17个字段，分析组写入 RFM_SCORE / RFM_TIER

## 五、上机后需要确认的字段

以下字段在文档中是推测值，**进系统后必须用 SE16N 验证再写代码**：

| 待确认项 | 查哪张表 | 查什么字段 | 影响哪个规则 |
|---------|---------|----------|------------|
| 销售订单类型 | `VBAK` | `AUART` 有哪些值 | 规则3 CDS View WHERE 条件 |

| 合同到期日位置 | `VEDA` | `VBDAT` 是否有数据 | 规则4 关键字段 |
| 返利协议类型 | `KONA` | `BOTYP` 有哪些值 | 规则5 CDS View WHERE 条件 |
| 公司代码 | `T001` | `BUKRS` 实际值 | 所有规则 |
| 销售组织 | `TVKO` | `VKORG` 实际值 | 规则3/4/5 |
