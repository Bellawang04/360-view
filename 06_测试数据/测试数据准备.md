# 测试数据准备

> 第1周任务：建6个测试客户，覆盖所有预警场景  
> 事务码：**XD01**（建客户）

---

## 操作步骤

### 建客户（所有客户第一步）
```
1. 事务码：XD01
2. 客户编号：填 Z_TEST_00x（或留空让系统分配）
3. 公司代码：CN01（按项目实际填）
4. 销售组织：CN01
5. 地址：随便填中国地址
6. 保存
```

### Z_TEST_001 — 造AR逾期数据
```
需要找财务顾问协助第一次操作
事务码 F-22（输入AR凭证）
- 凭证日期：设为90天前（如2026-03-13）
- 到期日：同样设为90天前
- 客户：Z_TEST_001
- 金额：50,000 CNY
保存后用 FBL5N 验证账龄
```

### Z_TEST_002 — 造信用超限
```
XD01 建客户时：
- "信用管理"页签 → 信用额度 = 1,000,000 CNY
用 VKM1 查信用状态，确认占用比例显示正确
（需要先有销售订单才会有占用额，可以用VA01建一张大额订单）
```

### Z_TEST_004 — 造合同到期
```
事务码：VA41（建合同）
- 合同类型：KM（框架合同，按实际调整）
- 客户：Z_TEST_004
- 有效期结束日：设为20天后（2026-07-02）
保存合同编号，后续用于验证 CONTRACT_DAYS 计算
```

## 验证方法

建完测试客户后，用以下方式验证数据：

| 验证什么 | 事务码/方法 |
|---------|-----------|
| AR账龄 | FBL5N（按客户查未清项） |
| 信用状态 | VKM1 或 FD32 |
| 销售订单 | VA05（按客户查订单列表） |
| 合同 | VA45（合同列表） |
| CDS View 计算结果 | SE16N 查 Z_CUST_KPI_CACHE |
