# 客户360视图 POC — 九号公司

> SAP S/4HANA 2024/2025 Private Cloud  
> 目标：给销售、财务、管理层在 SAP 系统里建一个客户全景页面

---

## 项目概览

| 项 | 内容 |
|----|------|
| 客户 | 九号公司（Ninebot） |
| 系统 | SAP S/4HANA 2024/2025 Private Cloud |
| 周期 | 4周 POC |
| 我的分组 | **智能提醒组**（2人） |
| 最终交付 | Customer 360° View — 打开任意客户，30秒内看到4个维度预警状态 |

---

## 两组分工

```
智能提醒组（我们）          智能分析组
─────────────────          ─────────────────
负责"亮红灯"               负责"分析客户"
5个自动检测规则             RFM客户评分模型
每日批处理 Job              6个SAP自带分析页面
写入 Z_CUST_KPI_CACHE  ←→  读取缓存表展示数据
4张预警卡片数据             Customer 360° 页面框架
```

## 关键接口约定（与智能分析组）

> 第2周会议确认，之后不再改字段名

共享缓存表：`Z_CUST_KPI_CACHE`  
主键：`MANDT + KUNNR + BUKRS + VKORG`  
提醒组写入：AR逾期/信用/活跃度/合同/返利 共17个字段  
分析组写入：`RFM_SCORE`、`RFM_TIER`
