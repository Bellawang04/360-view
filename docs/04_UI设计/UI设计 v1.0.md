# UI 设计方案 — 客户360视图 Phase 1
## 九号公司 SAP Fiori 界面设计规格

> 版本：v1.0 · 2026-06-10  
> 适用范围：Phase 1 原生能力验证（第1-4周）  
> 设计基础：SAP Fiori 3 Design System · S/4HANA 2024/2025

---

## 一、整体信息架构

```
Fiori Launchpad（角色主页）
│
├── [客户经理] My Sales Overview (F2200)          ← 智能分析组 开通配置
│   └── → 点击客户 → Customer 360° View (F2187A) ← 核心页面（两组共建）
│
├── [财务] Order to Cash Dashboard (F4663)        ← 智能分析组 开通配置
│   └── → 点击客户 → Customer 360° View (F2187A)
│
├── [管理层] Sales Management Overview (F2601)    ← 智能分析组 开通配置
│   └── → Customer Overview (F4645) → Customer 360° View (F2187A)
│
└── KPI Tiles（各角色专属）
    ├── 预警类 Tiles（AR逾期/合同到期/未下单）    ← 智能提醒组 负责数据
    └── 分析类 Tiles（RFM分层/客户价值）          ← 智能分析组 负责数据
```

**Customer 360° View 是所有路径的终点**，所有预警和分析结果都汇聚于此。

### 3.2 客户头部信息区

**状态徽章规格（Header Badge）：**

| 徽章 | 数据字段 | 颜色逻辑 | 点击行为 |
|------|---------|---------|---------|
| 财务风险 | `AR_STATUS` 最高等级 | 红=高危，橙=预警，绿=正常 | 展开财务风险雷达卡片 |
| 合规状态 | `CONTRACT_ST` / `REBATE_ST` 最高等级 | 红=逾期，橙=30天内，绿=正常 | 展开合规状态卡片 |
| RFM等级 | `RFM_TIER` | A=深绿，B=绿，C=橙，D=红 | 展开AI洞察卡片 |
| 工商状态 | `Z_CUST_RISK_EXT.RISK_LEVEL` | 红=高危，橙=关注，灰=无数据，绿=正常 | 展开外部风险标签卡片 |

**徽章 UI 规格：**
- 使用 SAP `sap.m.ObjectStatus` 控件
- 文字格式：`[图标] 字段名: 状态文字`
- 高危状态使用 `@UI.criticality: #NEGATIVE`（红色）
- 预警状态使用 `@UI.criticality: #CRITICAL`（橙色）
- 正常状态使用 `@UI.criticality: #POSITIVE`（绿色）

## 六、交互行为规范

### 6.1 卡片展开/折叠

- 每张自定义卡片默认显示摘要（3-4行关键数据）
- 右上角"展开详情 ↗"跳转至独立详情页或侧边板（Side Panel）
- 卡片之间无强依赖，独立加载，某张加载失败不影响其他卡片

### 6.2 数据加载状态

- 卡片数据来源于预计算字段：**首屏加载 < 1.5秒**（读 `Z_CUST_KPI_CACHE`，无实时计算）
- 月度趋势图表为实时查询，加载时显示 skeleton 骨架屏
- 如预计算数据超过24小时未刷新，显示黄色提示条："数据可能不是最新，最后刷新：{时间}"

### 6.3 空数据处理

| 场景 | 显示方式 |
|------|---------|
| 客户无合同记录 | 合规卡片显示"无合同记录"，灰色，不显示预警状态 |
| 外部风险无数据 | 外部风险卡片显示"暂无外部数据"，灰色 |
| Phase 2/3 功能 | 显示功能名称 + 灰色 "—— 即将上线 (Phase X)" |
| RFM计算中 | 显示"评分计算中"，spinner，下次批量后自动刷新 |

### 6.4 导出功能

- Customer Overview 列表页支持导出为 Excel（使用 SAP 标准 `Export` 按钮）
- 导出列包含：客户名称、客户编号、销售额、AR状态、信贷占用率、RFM等级、最近下单日期
