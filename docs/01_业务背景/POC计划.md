## 我们在做什么

给九号公司的销售、财务和管理层，在 SAP 系统里建一个「客户全景页面」。

打开任意一个客户，30秒内看到：
- 这个客户欠钱了吗？欠了多久？
- 他最近还在买东西吗？多久没下单了？
- 合同快到期了吗？
- 这个客户算高价值还是快流失？

---

## 两个小组

### 智能提醒组（2人）
> 负责"出问题就亮红灯"

做5个自动检测规则，系统每天自动扫描所有客户，把异常状态算好存起来：

| 检测什么 | 触发条件 |
| --- | --- |
| 应收账款逾期 | 超过30/60/90/180天 |
| 信用额度快用完 | 占用超过70% / 85% / 100% |
| 长期没有下单 | 超过60天 |
| 合同快到期 | 距到期15 / 30 / 60天 |
| 年度返利协议没签 | 新年度开始后未签署 |

---

### 智能分析组（2人）
> 负责"分析客户"

1. 开通6个 SAP 自带的分析页面（不用自己写，配置启用即可）
2. 做一个客户价值评分模型（RFM），把客户自动分成 A / B / C / D 四档
3. 给三类用户（销售 / 财务 / 管理层）配好各自的登录权限

**RFM 是什么：** 根据客户「最近买过吗」「买了几次」「买了多少钱」三个维度打分，综合算出 A（高价值）到 D（低活跃）的等级。

---

## 4周安排

### 第1周：把基础打通

- 分析组：开通 SAP 自带的4个客户分析页面
- 提醒组：开通 SAP 自带的2个合同/退货监控页面
- 两组一起：在测试系统里找好用于演示的样本客户（需要覆盖各种情况：有逾期的、快流失的、优质的）

### 第2周：核心逻辑开发

- 提醒组：写3个检测规则（账龄逾期 / 信用额度 / 长期未下单）
- 分析组：搭 RFM 评分的计算框架
- **两组一起开会**：确定「把算好的结果存在哪里」，之后两组都往同一个地方写，这一步必须对齐

### 第3周：补全 + 配权限

- 提醒组：补写2个规则（合同到期 / 返利未签），写每天自动跑的批处理程序
- 分析组：完成 RFM 分级逻辑，和业务同事核对分档标准是否合理，配完三类用户的登录权限

### 第4周：拼在一起 + 演示

- 把所有预警状态和 RFM 等级都嵌入客户主页面
- 准备3-5条假数据（模拟工商异常、司法执行），用于演示外部风险标签
- 选5个样本客户，走一遍完整演示流程，修正数据问题
- 准备演示脚本（三个角色各一条路径，每条不超过10分钟）

---

## 最终验收标准

演示时需要能走通这3条故事：

**销售视角**
打开某个客户，能看到：他欠了多少钱逾期多久、最近一次下单是多久之前、RFM 是几级、有没有外部风险

**财务视角**
打开财务工作台，能看到信用超限的客户清单，按逾期金额排序，点击直接进入客户详情

**管理层视角**
打开客户列表概览，能看到 A/B/C/D 分布，能筛选出 D 级客户，能导出名单
